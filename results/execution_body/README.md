# Execution-Body Result Artifacts

This folder contains the compact SpinMesh Runtime execution-body sweep generated by:

```bash
PYTHONPATH=src python tools/run_execution_body_experiments.py --output-dir results/execution_body
```

The fixed source problem is `frustrated_n6_j2_05_p2_seed314`: a six-spin `j1j2_frustrated` Hamiltonian at `J2/J1 = 0.5`, fixed magnetization `M = 0`, fixed QAOA depth `p = 2`, fixed seed, and frozen optimized angles. The sweep varies execution body variables only: topology, routing method, initial layout, transpiler level, calibration age, shot budget, session policy, and mitigation policy.

The key routing result is that the same source-level QAOA circuit produced execution-dependent physical-observable error. In the frozen-circuit routing sweep, transpiled depth ranged from `378` to `1568`, two-qubit gate count ranged from `96` to `400`, and two-qubit gate count correlated positively with correlation error (`r ~= 0.60`) while correlating negatively with valid-sector ratio (`r ~= -0.64`).

The key decision result is intentionally negative: the compact trust gate rejected all `90` quantum execution records, primarily because routing inflation was too high before calibration, shot, or mitigation criteria could rescue physical interpretability.
